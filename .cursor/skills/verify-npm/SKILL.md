# 第三方 npm API 稽核（本 repo 版本對照）

## 何時使用

- 使用者要求審閱／驗證某段程式是否與**本專案已安裝套件版本**一致。
- PR 前檢查：import、呼叫參數、預設行為、型別是否與 `node_modules` 相符。

若未指定檔案或片段，先請使用者提供路徑或貼上程式。

## 執行步驟

1. 從目標檔案列出所有 **外部 npm import**（排除 `@/` 與相對路徑專案內模組）。
2. 對每個套件：
   - 讀根目錄 `package.json` 取得**精確版本**（必要時對照 lockfile）。
   - 在 `node_modules/<name>/` 內定位 `types` / `main` 對應的 `d.ts` 或入口 `.js`。
   - 核對：**函式名稱、參數個數與順序、可選參數、回傳型別**是否與使用者程式一致。
   - 若行為依賴**預設參數或執行時選項**（例如「預設只回傳 text、不回傳 blocks」），且 d.ts 未載明，必須讀 JS 實作並註明依據行號或關鍵片段。
3. 標出：**確定正確**、**可能不相容**、**未能在本 repo 驗證**（並說明缺什麼：例如無法讀 node_modules）。

## 輸出格式（必須使用表格）

| 套件 | 專案版本 | 稽核項目                      | 本 repo 依據（路徑） | 結論               |
| ---- | -------- | ----------------------------- | -------------------- | ------------------ |
| …    | …        | 例如 `recognize` 第三參數預設 | …                    | OK / 需改 / 未驗證 |

最後一段用條列整理：**必修改項**、**建議項**、**無問題項**。

## 不要做的事

- 未讀本專案 `package.json` 與 `node_modules` 就斷言「官方預設一定是…」。
- 混用其他專案或舊版文件結論，且未標註與本專案版本差異。
- 除非使用者明確要求，否則只輸出稽核報告，不直接改檔。
